## 2. Git & GitHub 핵심 개념
- **Repository:** 코드를 저장하는 저장소 (예: `project.a`).
- **origin:** 내 로컬 저장소와 연결된 원격 서버(GitHub)의 별명.
- **main:** 프로젝트의 기본이 되는 줄기(브랜치).
- **HEAD:** 내가 현재 작업 중인 위치를 가리키는 포인터.
- **Fork:** 남의 저장소를 내 계정으로 통째로 복사 (저장소 외부의 복제). 오픈 소스 기여 시 사용.
- **Branch:** 한 저장소 내에서 독립적인 작업 공간을 분리 (저장소 내부의 가지). 기능 개발 및 실험 시 사용.

## 2-1. Git 핵심 개념 정복
- **Remote vs Local:** `origin`은 GitHub 서버의 별명. `commit`은 로컬 저장, `push`는 서버 저장.
- **HEAD & main:** `main`은 기본 줄기, `HEAD`는 나의 현재 작업 위치(포인터).
- **History 관리:** `git log`와 VS Code의 `Timeline` 패널을 통해 과거 코드 확인 및 비교 가능.

## 3. 주요 명령어 워크플로우
0. `git config --global user.name [name]`: user name 설정.
1. `git config --global user.email [e-mail]`: user email 설정.
2. `git clone [URL]`: 서버의 코드를 내 컴퓨터로 복제.
3. `git add .`: 변경 사항을 스테이징 영역(준비 단계)에 추가.
4. `git commit -m "메시지"`: 내 로컬 저장소에 기록을 확정.
5. `git push origin main`: 로컬 기록을 GitHub 서버로 전송.
6. `git branch [new-branch]`: new branch 만들기
7. `git checkout/switch [new-branch]`: 작업위치(HEAD)를 새 브랜치로 이동

## 4. 실전 워크플로우
- **ID 변경 대응:** GitHub 아이디 변경 시 `git remote set-url origin`으로 주소 갱신 필요.
- **반영 절차:** `add` (준비) -> `commit` (기록) -> `push` (전송).
